数据库 Something

2019-06-06

Contents

  • 主键
    1. 唯一标志表中唯一行，确保数据唯一性
    2. 可以方便地把一个表和另外一个表进行关联
  • 外键

存储引擎使数据库中的表可以用不同的方式存储 show engines

  • innoDB
    • 给MySQL提供事务支持、回滚、崩溃修复能力和多版本并发事务控制的事务安全。
    • 支持自动增长列AUTO_INCREMENT，自动增长列的值不能为空，而且必须唯一。MySQL规定自增列必须为主键。
    • 支持外键，外键所在表为子表，外键所依赖的表为父表。父表中被子表外键关联的字段必须为主键。
    • 缺点是读写效率稍差，占用数据空间相对较大
  • MyISAM
    • 支持三种不同存储格式，包括静态型、动态型和压缩型。静态型为默认存储格式，其字段长度固定，动态型包含边长字段，压缩型需要使用myisampack工具创建，占用磁盘空间小。
    • 优点：占用空间小，处理速度快
    • 缺点：不支持事务的完整性和并发性。
  • MEMORY
    • 使用存储在内存中的内容创建表，所有数据也存放在内存中。
    • 默认使用哈希索引，其速度比使用B树索引快。
    • 表的大小受到限制，参数max_rows可以在创建时指定，max_heap_table_size默认大小16MB，可以按需扩大。
    • 处理速度非常快，数据易丢失，生命周期短。

根据不同存储引擎的特点可以知道： + 当对事物完整性要求较高，要求实现并发控制或者需要频繁进行更新删除操作的数据库可以选择InnoDB + 如果表主要用于插入新纪录或者读取记录，或者应用的完整性和并发性要求较低可以选择MyISAM + 如果需要很快的读写速度，对数据的安全性要求不高可以选择MEMORY
